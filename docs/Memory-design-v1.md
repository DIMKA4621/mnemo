# Система пов'язування claude-mem пам'яті з агентами

## Область видимості
Основний принцип - це зберігати всі .md файли у репозиторії на рівні проекту.
Для файлів, які реалізовують нативний механізм пам'яті або які слугують у ролі пам'яті окремих агентів, бажано зберегти ідентичну структуру в .claude директорії проекту для забезпечення сумісності із глобальним форматом пам'яті на рівні системи.
Загальна памʼять - .claude/memory/ у форматі точно як нативна (MEMORY.md як індекс + logs/ + topic-файли).
Per-agent памʼять - .claude/agent-memory/<name>/ у тому ж нативному форматі.

## Розподілення пам'яті у векторній базі claude-mem
Головний принцип - це одна субд sqlite + chrome, яка в собі має розподілекння області видимості по проектам.
Область видимості проекту включає загальну пам'ять проекту, а також агентні блоки пам'яті по агентно (тих хто представлені у проекті).

## Запис або редагування пам'яті як файлів
Агенти використовують нативні інструменти для оновлення memory (як нативний механізм пам'яті агентів) та/або тулзи для редагування .md файлів (у ролі окремих від нативної memory файлів із пам'ятю)
Коли трігериться:
* хуки на cтоп сесію,
* прописані рулзи/інструкції які зобов'язують підтримувати меморі у актуальному стані, та/або у випадках значних змін проекту, після комітів і т.д.

## Оновлення (переіндексація) пам'яті як векторної бд claude-mem
Автоматизація на рівні хуків:
* хуки на оновлення memory (як нативний механізм пам'яті агентів),
* хуки на тулзи для редагування md файлів (у ролі окремих від нативної memory файлів із пам'ятю).
Концепція скрипту оновлення пам'яті:
* обходити всі наявні .md файли які слугують як пам'ять,
* переіндексовувати у векторну бд лише ті файли які змінилися.

## Читання пам'яті як векторної бд claude-mem
Основний принцип - це використоання інструментів/команд claude-mem для отримання релевантних знань з загальної пам'яті проекту та/або власних знань-приміток в межах цього проекту (у ролі особистої пам'яті агента, якщо реалізована як окремий скоуп пам'яті)
Коли трігериться:
* хуки на старт сесію,
* хуки перед будь якими обдумуваннями/діями або після того як юзер вводить наступне повідомлення

# Ролі агентів

## Claude Code консоль
Роль: тімлід, сам нічого не змінює не створює, головна задача це керувати командою агентів прописаних у проекті, запускати своїх тімейт агентів як команду
і працювати та комунікувати з нимим
Пам'ять: отримує результати із спільної пам'яті проекту, записує пам'ять як загальну пам'ять проекту

## Член команди агентів з проекту
Роль: комунікувати з тімлідом і іншими агентами, виконувати свою роботу у чітко окресленому домені (агент девелопер, агент тестувальник, агент аналітик,
...)
Пам'ять: отримує результати із спільної пам'яті проекта, можливо також отримує результати із своєї пам'яті як сутності (але все одно на рівні проекту), записує пам'ять як загальну пам'ять проекту та/або пам'ять на рівні свого агента.
