# Дизайн інсталяції mnemo

> Дизайн інсталяції та підключення. Доповнює `Memory-design-v2.md`

## Підключення проєкту

Основний принцип — підключення виконує Claude Code за визначеною процедурою: детерміновані кроки виконуються скриптами; зміни в уже наявних файлах проєкту — з показом і підтвердженням користувача. Кроки крос-платформні (Linux / macOS / Windows).

## Рівень системи

Основний принцип — рушій (середовище виконання, ембединг-модель, векторний індекс) встановлюється одноразово в домашній директорії користувача, спільний для всіх проєктів, оновлюється незалежно від них.

* Точка входу рушія — лише для автоматизації (хуки, MCP, інсталяційний скіл), не для людини напряму.
* Модель завантажується тільки явним прогрівом, ніколи неявно.
* Індекс — локальний, одноразовий, відтворюваний, не в git.

## Рівень проєкту

Основний принцип — усе, що потрібно проєкту для роботи з памʼяттю, лежить у самому проєкті під git і успадковується кожним, хто його клонує.

* Реєстрація MCP-інструмента памʼяті.
* Хуки: повний reconcile на старті сесії, інкрементально на зміні memory-файлу, авто-підкидання релевантного в контекст.
* Скелет памʼяті: тонкий індекс + директорія логів + topic-файли; агентні блоки по-агентно.
* Виклики у git-tracked налаштуваннях — портативні, без машинно-залежних шляхів.
